# GenAI Content Governance Pipeline

**Automated CI/CD guardrails for LLM-generated technical documentation.**

## Overview
As engineering organizations integrate Generative AI to accelerate documentation workflows, the risk of injecting hallucinated features, passive voice, and semantic "fluff" into production environments increases exponentially. 

This repository demonstrates an automated governance infrastructure. It utilizes a custom CI/CD pipeline to intercept, lint, and validate AI-generated pull requests before they can be merged into the `main` branch. 

## The Architecture
This pipeline treats documentation with the same operational rigor as software deployment. 

* **The Engine:** A GitHub Action (`.github/workflows/genai-guardrail.yml`) triggered on any Pull Request that modifies Markdown files.
* **The Linter:** Uses the [Vale](https://vale.sh/) command-line tool to parse the text against established corporate style guides (Microsoft Manual of Style).
* **The AI Guardrail:** A custom, deeply restrictive ruleset (`styles/GenAI/Fluff.yml`) designed specifically to identify the semantic markers of LLM outputs (e.g., words like "delve," "transformative," or "seamlessly").

## Business Value
This infrastructure shifts documentation validation left. By catching AI-generated technical debt in the PR phase rather than the publishing phase, this system allows a single technical writer to act as a Systems Governor. 

It enables service teams to utilize AI to reduce documentation turnaround times by 80% while strictly enforcing the enterprise quality bar without manual editorial bottlenecks.

## How It Works
1. A contributor opens a Pull Request containing a draft generated by an AI LLM tool.
2. The GitHub Action automatically provisions an Ubuntu runner and executes the Vale linter against the modified files.
3. If the linter detects AI filler language or structural violations, the pipeline fails the build.
4. An automated comment is posted directly on the PR, flagging the exact line of code that requires human intervention. The code cannot be merged until the issue is resolved.
